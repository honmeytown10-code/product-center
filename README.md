# 商品中心 (Product Center) 需求与架构文档

本文档记录了商品中心核心模块的业务逻辑与前端交互规范。

## 1. POS 端商品管理重构（沽清/上下架/做法）

### 1.1 架构与页面布局重构（单线降维策略）
**背景**：解决商家在开启“渠道独立库存”或“渠道分组”等复杂配置下，商品列表展示信息过载（如多态胶囊）、操作动线混乱的问题。

- **全局单渠道切换栏（最高优先级）**
  - **展示规则**：
    - 开启“全渠道统一库存”时：隐藏该导航栏，全局按统一状态展示。
    - 未开启全渠道时（无论是否开启渠道分组）：在页面顶部固定展示“当前操作渠道”下拉框（仅包含基础的单一渠道，如 POS、小程序堂食、美团等，**不包含分组**）。
  - **交互逻辑**：该导航栏为页面的“顶级单维筛选器”，左侧预警列表与右侧商品网格**必须严格跟随当前选中的单一渠道进行数据过滤与状态渲染**。分组逻辑仅在弹窗内的操作阶段生效。

- **搜索框位置跃升**
  - 将搜索框从右侧分类区提取至顶部的全局渠道导航栏右侧，强调“搜索结果属于当前选中渠道的全局动作”。

- **左侧预警列表统一降维与批量入口下沉**
  - **废弃多态胶囊**：左侧预警列表（今日已沽清/低库存预警、已下架商品、已禁用做法）不再混合展示多渠道的差异状态。仅展示商品在**当前选中渠道**下的确切库存与单一状态标签（当日沽清/长期沽清）。
  - **操作入口下沉**：三个页面的左侧预警列表，其“批量操作”按钮统一从列表顶部移至列表最底部，点击后底部呼出批量操作栏（全选、沽清/上架、取消），与右侧主列表底部的操作栏在视觉与交互上达成全局对称。

### 1.2 商品卡片展示与本地设置优化
- **图片展示开关迁移至设备本地配置**
  - 移除商品列表页内联的“图文模式”切换按钮。
  - 新增独立的「POS本地设置」页面，将“商品列表显示图片”作为设备级配置项持久化（LocalStorage）。POS 默认不展示图片。
- **卡片自适应布局**
  - 当开启图片展示时，为避免文案重叠与截断，卡片网格自动降列（如 5列 降为 4列），并保证卡片高度≥250px。
  - **状态角标优化**：将“部分售罄”、“已售罄”等异常状态统一以右上角浮动角标形式呈现，避免与多规格标签或商品名称重叠。

### 1.3 多规格商品沽清弹窗重构（防晕设计）
**背景**：原有的“批量向下同步”输入框对于门店收银员认知成本过高，且容易误操作。

- **引入 SPU / SKU 沽清模式切换器**
  - 在多规格沽清弹窗左侧面板，新增模式切换选项：
    - **按商品统一设置（默认）**：隐藏规格明细列表，仅展示一组大号输入框，输入的库存与补足数量将隐式同步覆盖该商品的所有规格。
    - **按规格独立设置**：展开下方“生效规格明细”列表，店员勾选目标规格后，直接点击该规格行，利用右侧数字键盘为其**独立设置**不同的库存数值（彻底移除该模式下的全局批量输入框）。
- **文案精准化（消除歧义）**
  - 当日沽清模式下，两个输入框文案明确为 **“今日剩余”** 和 **“次日补足”**。
  - 长期沽清模式下，保留单一输入框 **“剩余可售数量”**。

### 1.4 场景化帮助与安全防呆（降低培训成本）
- **引入图文并茂的“如何沽清”操作指南**
  - 在沽清弹窗左上角提供帮助入口，点击弹出《如何沽清商品》的场景化图文说明。
  - **场景拆解**：
    - 场景一：今日商品不卖了，明日需自动恢复售卖（当日沽清用法）。
    - 场景二：商品无限期停售，直到人工重新补货（长期沽清用法）。
    - 场景三：恢复已沽清商品的正常售卖（取消沽清流程）。
  - **要求**：说明中需配有直观的系统 UI 简化示意图，并高亮关键按钮与输入框。
- **取消沽清（恢复售卖）的二次确认**
  - 业务规则：取消沽清默认会将商品恢复为无限库存。
  - 交互要求：无论是在弹窗内点击“取消沽清”，还是在左侧列表点击“撤销”图标，均必须强制弹出二次确认对话框，确认后方可执行，严防误触导致超卖。
